# 用户手册

最后更新：2026-04-13

## 1. 产品定位

`Agent Studio` 当前是一套“对话驱动的桌面自动化工作台”。

你不需要先手工创建工作流。默认做法是：

1. 在聊天区直接说目标
2. 系统在内部创建并运行任务
3. 右侧工作区展示这个会话下的任务状态、Agent / Subagent 状态、内联确认和最近事件

## 2. 启动方式

在项目根目录运行：

```powershell
.\scripts\run.ps1
```

如需检查后端是否在线：

- `GET /api/health`

## 3. 当前界面说明

### 左侧：会话与聊天

左侧是主入口，包含：

- 会话列表
- 当前会话聊天历史
- 图片附件入口
- 目标输入框
- `Send`

推荐使用方式：

1. 直接在输入框写你想完成的目标
2. 如有需要，附上一张或多张图片
3. 点击 `Send`

当前“聊天即任务”已经接通。每次发送目标，系统都可以在内部创建一个新的任务并开始执行。

### 右侧：会话执行态

右侧只展示当前会话相关内容，不再要求你手工编排流程。

当前会显示：

- 当前会话下的任务标签
- 每个任务里的 Agent / Subagent 标签
- 任务最近进展
- 任务执行轨迹
- 高风险动作的内联确认区
- 最近事件

## 4. 聊天任务的默认行为

当前版本里，聊天触发的自主任务默认优先走本地模型：

- Provider: `Ollama`
- Model: `qwen3-vl:4b`

这意味着你可以直接用聊天发起本地多模态桌面任务，例如：

- “观察当前桌面，告诉我下一步该点哪里”
- “分析这张截图里的错误信息”
- “继续完成这个桌面操作，必要时再问我确认”

## 5. 图片附件

聊天区支持附图。

你可以：

- 点击 `Add Images` 选择本地图片
- 点击 `Clear Images` 清空当前附件

图片会一起发送给模型，并且也会作为当前会话的最新视觉上下文供后续步骤复用。

## 6. 设置页

右上角 `Settings` 可配置：

- 主模型 Provider
- Model
- Base URL
- API Key
- Organization
- 桌面控制模式
- 审批超时时间
- 审批超时默认动作
- 界面语言
- 输出细节模式

设置页也支持：

- 单独测试当前 Provider 连通性
- 一次性检查全部 Provider 连通性
- 查看各 Provider 能力摘要

## 7. 高风险动作确认

当前高风险动作不会静默执行，而是以内联确认的方式出现在主界面右侧。

确认区支持三种操作：

- `Allow`
- `Deny`
- `Extra Prompt`

同时支持：

- 60 秒倒计时
- 超时后的默认动作
- 通过设置页切换超时默认同意 / 默认拒绝 / 默认提示词

## 8. 当前支持的核心能力

系统内部已经具备这些能力：

- 多模型路由
- 本地多模态
- 会话持久化
- 截屏
- OCR
- 文本定位
- 桌面控制授权
- 任务 / Agent / Subagent 递归执行
- 高风险脚本审查与批准执行

这些能力现在更多作为系统内部执行机制存在，而不是要求你手工操作它们。

## 9. 当前限制

当前版本仍处于从“工作流原型”向“对话式 Agent 工作台”过渡的阶段，所以还有这些限制：

- `/api/chat` 的模型路由还在继续收口
- 后端 fail-fast 行为还会继续加强
- 权限审计落库还没有完全闭环
- 暂停 / 继续 / 终止 / 人工接管仍在后续阶段中
