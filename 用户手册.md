# 用户手册

最后更新：2026-04-13

## 0. 维护约定

- 每次修改后都要同步更新 `checklog.md`。
- 每次影响产品入口、交互方式或能力范围的修改，都要同步更新本手册。

## 1. 产品说明

`Agent Studio` 是一个跨平台桌面自动化原型，当前已经具备这些基础能力：

- 桌面 UI + 本地 FastAPI 后端
- 多模型路由：`Mock`、`OpenAI Compatible`、`Ollama`
- 本地多模态：支持 `Ollama / qwen3-vl:4b`
- 会话持久化
- 截屏、OCR、文本定位
- 桌面控制授权
- 任务 / Agent / Subagent 递归执行
- 高风险动作审批

当前项目正在从“工作流式自动化原型”调整为“对话驱动的桌面自动化工作台”。

这意味着：

- 聊天入口会逐步成为默认主入口
- workflow / task / step 会继续存在，但更多作为内部执行机制
- 主界面会持续弱化手工编排入口

## 2. 启动方式

在项目根目录运行：

```powershell
.\scripts\run.ps1
```

如果只想确认后端是否在线，可以访问：

- `GET /api/health`

## 3. 模型设置

点击主界面右上角 `Settings` 可以配置：

- 主模型 provider
- 模型名
- Base URL
- API Key
- 输出模式
- 桌面控制模式
- 审批超时与默认动作

本地多模态推荐配置：

- Provider: `Ollama`
- Base URL: `http://127.0.0.1:11434`
- Model: `qwen3-vl:4b`

## 4. 当前使用方式

### 4.1 对话

你可以在聊天区直接输入目标，例如：

- “分析这个界面现在应该点哪里”
- “读取截图中的文字并告诉我下一步”
- “帮我完成一个本地桌面任务”

聊天区支持附加图片。

### 4.2 任务 / Agent

当前版本仍然保留任务、Agent、Subagent 的执行能力，用于：

- 自动规划下一步动作
- 递归分派子 Agent
- 记录执行结果
- 暂停在高风险动作处等待确认

这部分能力后续会继续保留，但会逐步从“用户手工编排”收敛为“系统内部执行状态”。

### 4.3 感知与动作

当前支持的核心工具能力包括：

- `Capture Screen`
- `Run OCR`
- `Find Text`
- `Analyze Image`
- `Move Mouse`
- `Left Click`
- `Type Text`
- `Execute Script`

## 5. 安全与审批

高风险动作不会直接静默执行。

当前保留的安全机制包括：

- 桌面控制授权模式
- 高风险脚本审批
- 审批超时默认策略
- 额外提示词 / 人工干预

后续这些能力会继续保留，但会更深地嵌入对话流程，而不是依赖独立流程设计界面。

## 6. 当前开发重点

当前优先级不是继续扩展工作流编辑能力，而是：

1. 清理 UI 合并残留和重复定义
2. 统一前后端接口契约
3. 把产品入口彻底切到“聊天即任务”
4. 让自动化围绕 conversation / session 组织
5. 让 workflow 对普通用户尽量隐身
