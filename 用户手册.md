## 1. 产品概览

`Agent Studio` 是一个跨平台桌面 Agent 工作台，当前已经具备这些能力：

- 桌面 UI + 本地 FastAPI 后端
- 模型路由：`Mock`、`OpenAI Compatible`、`Ollama`
- 本地多模态：支持 `Ollama / qwen3-vl:4b`
- 设置页支持全量 provider 连通性检查，包含本地 `Ollama`
- 多轮聊天与会话持久化
- 截屏、OCR、文本定位
- 桌面自动化控制授权
- 任务 / Agent / Subagent 递归编排
- 父 Agent 可以为子 Agent 分配不同模型，不必和父 Agent 使用同一路由
- 高风险脚本与高风险桌面控制的内联审批
- 自主 Agent：不必手工预先列步骤

## 2. 启动方式

建议先准备：

- 项目虚拟环境 `.venv`
- 本地已安装 `Ollama`
- 本地已拉取模型 `qwen3-vl:4b`

启动应用：

```powershell
.\scripts\run.ps1
```

如需确认后端是否在线：

- `GET /api/health`

## 3. 模型配置

点击右上角 `Settings` 可以配置：

- `Mock`
- `OpenAI Compatible`
- `Ollama`

本地多模态推荐：

- `Base URL`: `http://127.0.0.1:11434`
- `Model`: `qwen3-vl:4b`

设置页还支持：

- 输出模式：`仅最终结果` / `中间步骤摘要`
- 审批超时时间，默认 `60` 秒
- 审批超时默认动作：`同意` / `拒绝` / `额外提示词`
- 审批超时默认提示词内容

设置页里的模型检查分两种：

- `Test Connection`：只检查当前表单里的单个 provider
- `Test All Providers`：同时检查 `Mock`、`OpenAI Compatible`、`Ollama`，并返回每条路由的连通性、已发现模型、以及当前模型是否可用

## 4. 多模态聊天

聊天区支持：

- 文本输入
- 选择一张或多张图片
- 图文混合发送
- 纯图片发送
- 直接从聊天目标启动本地自主任务

统一接口：

- `POST /api/chat`

如果当前 provider 为 `Ollama` 且模型为 `qwen3-vl:4b`，就会走本地视觉理解链路。

聊天输入区现在有两个动作：

- `Send`：走普通聊天
- `启动本地 Agent 任务`：把当前输入直接创建成自主任务，并默认把根 Agent 绑定到本地 `Ollama / qwen3-vl:4b`

如果你在聊天区附带了图片再启动本地任务，系统会自动把这些图片转成 `Analyze Image` 种子步骤，然后继续交给自主 Agent 规划后续动作。

## 5. 任务、Agent 与自主执行

### 5.1 两种任务方式

当前支持两种方式：

- 手工步骤模式：先添加步骤，再运行
- 自主 Agent 模式：只写目标/指令，让模型自己决定下一步动作

你现在不需要人为预先列好步骤。步骤区仍然保留，作为可选的“种子步骤”。

### 5.2 支持的动作能力

自主 Agent 和手工步骤共用同一套能力：

- `Detect System`
- `Capture Screen`
- `Run OCR`
- `Find Text`
- `Analyze Image`
- `Execute Script`
- `Move Mouse`
- `Left Click`
- `Type Text`

其中自主 Agent 还支持一个内部动作：

- `Delegate Agent`

也就是当前 Agent 可以把一个子目标分配给新的 Subagent。

### 5.3 自主 Agent 的基本用法

在任务区里：

1. 填写 `Task Title`，可选
2. 在 `Goal / Instruction` 中描述目标
3. 保持 `Autonomous Agent` 勾选
4. 设置 `Max Planning Turns`
5. 直接点击 `Create Task`

如果不填标题，系统会自动根据指令生成标题。

除了任务区以外，你也可以直接在聊天输入区完成这件事：

1. 在聊天区输入一句目标
2. 可选地附带一张或多张图片
3. 点击 `启动本地 Agent 任务`

系统会自动：

- 创建任务
- 把根 Agent 默认绑定到本地 `qwen3-vl:4b`
- 立刻启动任务
- 自动切换到任务标签页查看执行结果

### 5.4 种子步骤

即使使用自主 Agent，你仍然可以先加一些种子步骤。例如：

- 先 `Capture Screen`
- 再把后续判断交给模型

也就是说：

- 手工步骤不是必须
- 但仍然可以用来给模型一个更稳定的起点

### 5.5 Agent / Subagent

当前支持：

- 给任务添加新的 Agent
- 给任意 Agent 添加 Subagent
- 自主 Agent 在运行中自己继续分配新的 Subagent
- 父 Agent 可以在委派时给子 Agent 指定自己的 `provider / model / base_url`

执行时会按 Agent 树递归运行。

这意味着：

- 根 Agent 可以继续使用当前默认模型
- 某个 Subagent 可以被单独分配到本地 `Ollama / qwen3-vl:4b`
- 另一个 Subagent 也可以继续走远端 `OpenAI Compatible`

每个 Agent 标签页里都会显示它当前绑定的模型路由，以及父 Agent 给它分配该模型的原因（如果有）。

### 5.6 任务与 Agent 标签页

任务区已经改成主界面标签页可视化：

- 外层标签页：每个已打开任务一个标签
- 内层标签页：`Overview` + 各个 `Agent / Subagent`

不需要额外窗口。

## 6. 感知与控制

这些功能都在主界面内完成，不需要额外弹窗：

- 截屏
- OCR
- 文本定位
- 任务 / Agent / Subagent 可视化查看

桌面控制权限支持：

- `Deny`
- `Ask Every Time`
- `Allow This Session`
- `Always Allow`

当模式为 `Ask Every Time` 时：

- 鼠标和键盘控制不会直接执行
- 会进入任务区的待审批卡片
- 由你选择 `同意`、`拒绝` 或 `额外提示词`

## 7. 高风险审批

当前主要高风险动作包括：

- 任意脚本执行
- 真实桌面控制动作

### 7.1 审批方式

现在审批全部改成主界面内联：

- 不再弹额外确认窗口
- 直接在任务区或系统区显示审批卡片

三种决策：

- `同意`
- `拒绝`
- `额外提示词`

### 7.2 60 秒超时

默认审批时间为 `60` 秒，倒计时会显示在主界面里。

超时后的默认动作可在 `Settings` 中切换：

- `同意`
- `拒绝`
- `额外提示词`

如果默认动作为 `额外提示词`：

- 当前高风险动作不会直接执行
- 系统会把默认提示词写回工作流上下文
- 后续模型步骤可以继续参考这条指导

### 7.3 额外提示词的用途

适合这类情况：

- “不要执行删除，先做检查”
- “不要立刻点击，先截图确认”
- “先给出安全替代方案”

## 8. 图片进入任务的方法

这些步骤可以直接绑定图片：

- `Run OCR`
- `Find Text`
- `Analyze Image`

方式：

- `Choose Image`
- `Use Latest Capture`

## 9. API 用法

### 9.1 健康检查

```http
GET /api/health
```

### 9.2 多模态聊天

```json
{
  "message": "请描述这张图中的界面，并告诉我下一步该点击哪里",
  "conversation_id": null,
  "attachments": [
    {
      "name": "screen.png",
      "media_type": "image/png",
      "image_path": "C:\\Users\\wyl21\\Desktop\\Agent\\data\\captures\\screen.png"
    }
  ]
}
```

### 9.3 检查全部 provider 连通性

```http
POST /api/provider/health/all
```

示例请求：

```json
{
  "provider": "openai_compatible",
  "base_url": "https://api.openai.com/v1",
  "api_key": "sk-...",
  "model": "gpt-4.1-mini",
  "organization": null,
  "timeout_seconds": 5.0,
  "allow_mock_fallback": false
}
```

返回结果会包含：

- 当前主路由
- 每个 provider 的连通状态
- 当前模型是否可用
- 已发现模型列表

### 9.4 创建自主任务

```json
{
  "title": "Open Settings",
  "instruction": "打开设置界面并完成确认操作",
  "autonomous": true,
  "max_iterations": 8,
  "steps": []
}
```

### 9.5 创建带种子步骤的自主任务

```json
{
  "instruction": "先观察当前页面，再判断是否需要点击设置按钮",
  "autonomous": true,
  "max_iterations": 8,
  "steps": [
    {
      "kind": "capture_screen"
    }
  ]
}
```

### 9.6 创建带模型分配的子 Agent

```json
{
  "name": "Vision Agent",
  "parent_agent_id": "agent-123",
  "instruction": "查看当前界面并总结下一步要点",
  "autonomous": true,
  "max_iterations": 4,
  "steps": [],
  "model_assignment": {
    "provider": "ollama",
    "model": "qwen3-vl:4b",
    "base_url": "http://127.0.0.1:11434",
    "assignment_reason": "使用本地多模态模型做界面理解"
  }
}
```

### 9.7 运行任务

```http
POST /api/tasks/{task_id}/run
```

### 9.8 处理待审批步骤

```http
POST /api/tasks/{task_id}/approve
```

请求体支持三种决策：

```json
{
  "decision": "allow",
  "extra_prompt": null
}
```

```json
{
  "decision": "deny",
  "extra_prompt": null
}
```

```json
{
  "decision": "prompt",
  "extra_prompt": "不要运行脚本，先输出安全替代方案"
}
```
