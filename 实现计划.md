# 实现计划

## 1. 当前已完成

- 已创建跨平台桌面骨架，采用 `PySide6 + FastAPI + httpx`
- 已配置项目级 `.venv`、安装脚本和启动脚本
- 已实现基础桌面界面
- 已实现本地后端服务与健康检查接口
- 已实现模型 provider 抽象
- 已接入三类模型入口：
  - `mock`
  - `openai_compatible`
  - `ollama`
- 已实现桌面控制权限模式骨架：
  - `deny`
  - `ask`
  - `allow_session`
  - `allow_always`
- 已实现安全的 demo 控制器，当前只记录动作，不真正注入鼠标键盘事件

## 2. 目标产品拆分

### 阶段 A：可用 MVP

目标：把当前骨架做成一个可演示、可操作、可切换模型来源的桌面产品。

工作项：

- 补充设置页
  - 存储 API Key、Base URL、默认模型
  - 增加本地模型连接测试
- 补充聊天会话能力
  - 多轮上下文
  - 会话历史
  - 会话命名和归档
- 增加任务面板
  - 任务创建
  - 任务状态
  - 执行日志
- 增加本地持久化
  - 推荐 `SQLite`
  - 保存设置、会话、任务、授权记录

验收标准：

- 能在桌面端切换 OpenAI 兼容接口和 Ollama
- 能保存设置并在重启后恢复
- 能看到清晰的日志和错误提示

### 阶段 B：真实桌面控制

目标：把当前 demo 控制器替换为真实的跨平台输入控制层。

工作项：

- 定义统一控制接口
  - 鼠标移动
  - 点击
  - 双击
  - 键盘输入
  - 快捷键
- 实现平台适配器
  - Windows
  - macOS
  - Linux
- 增加授权弹窗
  - 首次授权
  - 单次授权
  - 会话授权
  - 永久授权
- 增加安全机制
  - 紧急停止快捷键
  - 用户输入抢占即暂停
  - 执行前倒计时
  - 审计日志

建议实现方式：

- 先做 `InputController` 的多平台实现
- 再做授权弹窗和执行编排
- 最后接真实动作回放

### 阶段 C：感知与自动化执行

目标：让软件不只是“能点”，而是“知道该点哪里”。

工作项：

- 窗口和应用识别
- 截图能力
- OCR 文本识别
- UI 元素定位
- 视觉匹配兜底
- 自动化工作流引擎
  - 步骤定义
  - 条件判断
  - 重试
  - 回滚

建议优先级：

1. 系统控件树 / 可访问性接口
2. OCR
3. 图像模板匹配
4. 大模型辅助决策

### 阶段 D：模型编排

目标：把“大模型 API”和“本地模型部署”真正变成可切换、可扩展的能力层。

工作项：

- 增加 provider 注册中心
- 增加模型能力描述
  - 文本
  - 视觉
  - 工具调用
  - 上下文长度
- 增加 provider 健康检查
- 增加自动回退策略
  - 首选云端
  - 本地兜底
  - mock 开发模式
- 增加统一响应规范

建议兼容目标：

- OpenAI 兼容 API
- Ollama
- LM Studio 的 OpenAI 兼容接口
- 其他本地 OpenAI 兼容推理服务

### 阶段 E：工程化与发布

目标：让项目能稳定开发、测试、打包、交付。

工作项：

- 数据库迁移
- 配置加密
- 日志归档
- 自动化测试
  - 单元测试
  - API 测试
  - UI smoke 测试
- 打包与发布
  - Windows 安装包
  - macOS App
  - Linux 包
- 崩溃恢复和诊断信息

## 4. 近期执行顺序

建议按下面顺序推进：

1. 增加 `SQLite` 持久化层
2. 完成会话历史和设置存储
3. 加入真正的 provider 连通性检测
4. 接入真实桌面控制适配器
5. 增加截图、OCR、元素定位
6. 实现工作流编排与任务执行
7. 最后再做安装包和跨平台发布细节

## 5. 风险与注意事项

- 桌面控制跨平台差异很大，建议先把 Windows 做深，再补 macOS 和 Linux
- 本地模型能力差异明显，必须做 provider 抽象，不能把某一家接口写死
- 权限控制必须默认保守，不能默认允许后台操控鼠标键盘
- 自动化不要一开始就全靠视觉匹配，优先系统级可访问性接口

## 6. 当前推进进度

- 阶段 A：已完成
- 阶段 B：已完成 Windows 第一版真实输入控制
- 阶段 C：已完成截图、OCR、文本定位与工作流执行
- 阶段 D：已完成 provider 能力描述、回退策略与统一模型路由
- 阶段 F / 第 7 步：已完成“聊天目标直达本地自主任务”快捷入口

当前新增进展：

- 自主 Agent 不再依赖预设步骤
- 父 Agent 可以给子 Agent 指定不同的 `provider / model / base_url`
- 设置页支持检查全部 provider 连通性，包含本地 `Ollama`
- 子 Agent 的模型分配结果会显示在任务标签页里
- 聊天输入区支持一键创建并启动默认走本地模型的自主任务
