## 1. 产品方向

项目已经正式从“工作流编排 / 手工定义任务流原型”转向“以对话为核心入口的桌面自动化 Agent 工作台”。

当前默认原则：

1. 用户通过聊天直接表达目标，不再以手工创建 workflow / step / task tree 作为主入口
2. workflow、task、agent、approval 继续保留，但主要作为后端内部执行状态机
3. 系统围绕 conversation / session 组织自动化，不围绕 workflow definition 组织
4. 高风险动作仍然保留确认与审查能力，但要以内联确认的方式嵌入对话流程
5. 主界面只保留聊天、当前会话执行态、Agent 状态、事件和必要确认入口

## 2. 当前主链路

后续开发继续优先收敛到这 6 条主链路：

1. `chat / session`
2. `agent orchestrator`
3. `perception`
4. `desktop controller / system executor`
5. `memory / persistence`
6. `permission & safety`

当前不再优先投入：

- 面向普通用户的 workflow editor
- 通用 BPM / 流程引擎式产品形态
- 让用户维护节点、分支、审批图
- 以脚本编写作为主要使用方式

## 4. 阶段 H：对话式 Agent 工作台

目标：彻底完成“聊天即任务、任务围绕会话、workflow 对用户隐身”的产品形态。

核心工作：

1. 建立真正的 conversation-centered agent loop
2. 让模型自己决定何时截屏、OCR、定位、控制桌面和发起子 Agent
3. 让失败重试、卡点说明、人工接管和继续对话修正形成一个闭环
4. 将高风险确认完全嵌入聊天与执行流，不再依赖旧流程式界面

验收标准：

- 用户只通过聊天就能发起并完成一个桌面任务
- 用户不需要手工创建 workflow
- 系统能连续执行多个感知 / 动作回合
- 高风险动作会暂停并请求确认
- 用户能在一个会话里看到完整执行过程，并继续修正任务
